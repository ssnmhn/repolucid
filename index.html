Questions you should ask

My code is private. What touches it?

A read-only clone on infrastructure we control, used solely to produce your pack, deleted after delivery. Your code is never used to train anything. We'll sign your NDA. If even that's too much, start with a public repo or a representative subtree.

How is this different from the free agent-readiness scanners?

Scanners check whether files exist; some auto-generate boilerplate. We read the codebase and write the content — the architecture, the invariants, the sharp edges. Compare any scanner's output to the samples above; that difference is the product.

Why would I trust AI-written docs?

Three reasons, in escalating order: every claim cites the code it describes, so your review is fast; a human editor verifies the pack before it ships; and you pay only if you merge. If it's wrong, it costs you one code review's worth of time and $0.

What stacks do you cover?

Any language a frontier model reads well — which in 2026 is effectively all of them. Monorepos are fine. Above ~300k LOC the Sprint becomes a scoped quote, still flat-fee.

48 hours, really?

Yes — the reading is machine-speed; the human review is the schedule constraint. Clock starts when we get access, and the free mini-audit usually lands same-day.
